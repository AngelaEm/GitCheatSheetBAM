git gc

Står för "garbage collect". Rensar upp onödiga filer för att optimera det lokala git-repot. Git sparar en massa data när man arbetar och det kan bli ganska mycket. Efter ett tag är en del data inte längre nödvändig eller användbar och då kan man rensa med git gc. Kan vara lurigt att använda om man inte har full koll på vad som rensas bort, speciellt om man delar repo med andra.

git diff [first commit] [second commit]

Listar ändringar i filerna mellan first commit och second commit så man kan se skillnaderna mellan två commits.

git tag

Används för att märka specifika commits med enkla namn. Detta är bra så det är lättare att hitta viktiga commits.

git push -all

Skickar alla lokala brancher till remote repot. Användbart om man vill pusha flera branches på en gång.

git show

Visar detaljerad information om den senaste commiten. Visar meddelandet och diffen för ändringarna.

git show [commit]

Visar detaljerad information om en specifik commit i stället för den senaste. Visar meddelandet och diffen för ändringarna.

git revert [commit]

Skapar en ny commit som ångrar effekten av tidigare commit men bevarar projektets historia. Det ser ut som att det aldrig var något misstag, men man kan fortfarande se alla steg, inklusive misstaget och hur man fixade det.

git push -u origin main

Används för att skicka ändringar från ett lokala repo till ett remote repo. Push används för att skicka ändingar. -u står för "upstream" och används för att sätta remotebranchen som standard så att man vi framtida pushar bara behöver skriva git push. Origin är standardnamnet på det remote repot som det lokala repot är kopplat till. Main är dit man skickar ändringarna.

git -version

Visar vilken version av Git som man har installerad.

git stash

Används för att tillfälligt lagra ändringar som man inte är redo att göra en commit av. Man kan spara sitt arbete utan att göra en commit. Kan vara användbart om man behöver jobba i olika branches men inte vill förlora eller commita det man gjort.

git switch [branch-name]

Byter till branchen [branch-name].

git switch -c [new-branch-name]

Skapar en ny branch [new-branch-name] och byter till den direkt. C står för create.
