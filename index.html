git commit -m "[message]"

Kommandot används för att göra en "commit" till ett Git-repo. Den används för att spara ändringarna som man gjort till det lokala "commits" inför push.

git restore [file]

Kommandot används för att återställa filen i ditt Git-repo till dess senaste sparade tillstånd.

git push

Kommandot används för att skicka (pusha) dina lokala commits till en fjärrserver (remote repository).

git pull

Kommandot används för att hämta (fetcha) och integrera (merge) ändringar från en fjärrserver (remote repository), till din lokala arbetskopia av ett Git-repo

git rm [file]

Kommandot används för att ta bort (remove) en fil från både ditt arbetsområde och från Git-repo.

git branch

Kommandot används för att lista grenar i ett Git-repo.

git branch -a

Kommandot används för att visa en lista över både lokala grenar (branches) och fjärrgrenar (remote branches) i ett Git-repo.

git branch -r

Kommandot används för att visa en lista över fjärrgrenar (remote branches) i ett Git-repos.

git checkout -b [new-branch-name]

Kommandot används för att skapa och byta till en ny gren (branch) i ett Git-repo i ett enda steg.

git log

Kommandot används för att visa en historik över commits i ett Git-repo.

git fetch

Kommandot som hämtar alla ändringar från en fjärrserver (remote repository) till ditt lokala Git-repo.

git tag

Kommandot används för att märka specifika punkter i historiken för en kodbas, vanligtvis för att markera speciella versioner som släpps.
